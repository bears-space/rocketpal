# How to contribute to RocketPal

Thank you for your interest in contributing to RocketPal! ✨

Before contributing to the project, please read and acknowledge the community [Code of Conduct](CODE_OF_CONDUCT.md).

## Reporting Issues

- Before reporting an issue, please check that it was not already reported by searching on GitHub under [Issues](https://github.com/bears-space/rocketpal/issues).
- If you're unable to find an open issue, [open a new one](https://github.com/bears-space/rocketpal/issues/new). Be sure to include a **clear title and description**, as much relevant information as possible, and ideally a code sample demonstrating the unexpected behavior.
- Use the provided issue templates. Feel free to deviate from them if it helps bring the point across.

## Code Contributions

- Note that **changes that are purely cosmetic** in nature and do not add anything to the functionality, maintainability, stability, testability, documentation quality etc. to the project **will generally not be accepted**. This is done to prevent spam (e.g. pure whitespace changes etc.) from giving people contributor status.
- All code contributions shall happen via GitHub [Pull Requests](https://github.com/bears-space/rocketpal/pulls). When your code is ready, [open a new one](https://github.com/bears-space/rocketpal/pulls/compare).
- If you are a BEARS member, you can work within branches.
- If you are an external contributor, you can work within your own personal fork of the repository and contribute to the main repository via pull requests.
- Make sure your code passes all unit tests (`uv run python -m pytest`).
- Make sure you have applied all pending pre-commit changes (`pre-commit run`).
- Please add yourself to the `AUTHORS` file. Ideally, your git contribution email (`git config user.email`) should match what you add to the file.
